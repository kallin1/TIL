## [2026-03-30] ERD 테이블 및 컬럼 명명규칙

📝 정보기록
### 개념 요약
#### 1. 외래 키(FK) 참조 및 컬럼명 설정
참조 관계: users 테이블의 id(PK)를 다른 테이블에서 user_id(FK)로 참조하는 것은 RDBMS의 표준적인 설계 방식임. PK와 FK의 컬럼명이 서로 달라도 상관없으나 이 경우 JOIN 쿼리 작성 시 컬럼명을 명시적으로 연결해줘야 함.

보내는 사람(sender_id), 받는 사람(receiver_id)처럼 의미를 명확히 해야 할 때는 이름을 다르게 짓는 것이 더 좋음.

주의사항: 연결되는 두 컬럼의 데이터 타입은 일치시켜야 한다.


#### 2. 네이밍 컨벤션: 스네이크 케이스 (snake_case)
userId (카멜 케이스) 대신 **user_id (스네이크 케이스)**를 사용하는 것이 관례.

대다수의 DB가 대소문자를 구분하지 않거나 설정에 따라 달라지기 떄문이다.
PostgreSQL 같은 DB는 따옴표 없이 쓴 식별자를 소문자로 자동 변환하기 때문에, userId로 만들어도 userid로 저장되어 의도와 달라질 수 있음.

장점 : 언더바(_)를 사용하면 어떤 환경에서도 단어 구분이 명확함. DB 환경에서 안전하고 예측 가능함


#### 3. 테이블 네이밍 규칙
컬럼명과 마찬가지로 스네이크 케이스 사용.

복수형을 사용하는 경우가 많지만 단수형을 사용하는 경우도 있다.
중요한건 프로젝트 전반에서 일관성을 유지하는 것.

연결 테이블: 다대다 관계를 잇는 테이블은 user_product와 같이 두 테이블의 이름을 조합하여 명명함.
